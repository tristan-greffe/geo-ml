# Introduction au Machine Learning : utilité et fonctionnement

Bienvenue dans cette section consacrée au **machine learning**.
Nous allons répondre à des questions essentielles : pourquoi utiliser le machine learning, à quoi il sert, et comment il fonctionne.

---

## Qu’est-ce que le machine learning ?

Le **machine learning** (ou apprentissage automatique) est l’étude d’**algorithmes informatiques qui s’améliorent automatiquement grâce aux données**.

Contrairement aux algorithmes classiques, où un humain doit définir toutes les règles et décisions, le machine learning :

* apprend à partir des **données elles-mêmes**,
* déduit la meilleure façon de résoudre un problème sans intervention humaine directe,
* fait partie de l’**intelligence artificielle**.

En résumé, un algorithme de machine learning n’est **pas programmé explicitement** pour chaque décision. Il apprend quelles actions sont optimales à partir des données existantes.

---

## Quels problèmes le machine learning peut-il résoudre ?

Le machine learning est très flexible et peut résoudre de nombreux types de problèmes, par exemple :

1. **Évaluation du risque client** :

   * Dans une banque, prédire si un client est solvable pour un prêt ou déterminer sa limite de crédit.

2. **Risque d’assurance** :

   * Estimer la probabilité qu’un client fasse une réclamation et déterminer le montant de sa prime.

3. **Prédiction de prix** :

   * Par exemple, prédire le prix de vente d’une maison à partir de ses caractéristiques (surface, nombre de chambres, type de sol…).

4. **Filtrage du spam** :

   * Identifier automatiquement les emails indésirables, améliorant considérablement l’efficacité du tri des messages.

5. **Segmentation de clients** :

   * Regrouper automatiquement des clients selon leurs comportements ou caractéristiques, utile pour le marketing ciblé.

> Ce ne sont que quelques exemples parmi une infinité d’applications possibles.

---

## Structure d’un problème de machine learning

Tout problème de machine learning repose sur deux notions clés :

* **Features / caractéristiques** : les attributs ou informations disponibles pour chaque donnée.
  Exemple : pour une maison → surface, nombre de chambres, type de sol, etc.

* **Labels / étiquettes** : ce que l’on souhaite prédire ou attribuer à chaque donnée.
  Exemple : le prix de vente d’une maison.

Le rôle d’un algorithme de machine learning (souvent appelé **estimateur**) est de **prévoir ou estimer la valeur du label à partir des caractéristiques**.

---

## Comment le machine learning fonctionne ?

Les algorithmes de machine learning utilisent des méthodes statistiques pour **déterminer automatiquement quelles caractéristiques sont importantes** pour prédire un label.
L’humain n’a plus besoin de définir manuellement l’importance de chaque attribut.

### Exemple concret : prédiction du prix d’une maison

1. Les caractéristiques connues :

   * Surface en m²
   * Nombre de chambres et salles de bain
   * Type de sol
   * Etc.

2. L’objectif : prédire le **prix de vente**.

Avant le machine learning, un humain devait **attribuer manuellement un poids à chaque caractéristique** pour estimer le prix.
Avec le machine learning, l’algorithme apprend **automatiquement la relation entre les caractéristiques et le prix**, à partir des données historiques.

---

## Pourquoi utiliser le machine learning ?

Le machine learning est indispensable pour traiter :

* Des problèmes **trop complexes** pour être codés manuellement.
* Des volumes de données **trop importants** pour une analyse humaine.
* Des tâches qui nécessitent de la **prédiction ou de l’automatisation**.

Exemples : filtrage du spam, lecture d’adresses manuscrites par les services postaux, reconnaissance de l’écriture, prédiction financière, etc.

---

## Limitations et conditions d’utilisation

L’efficacité du machine learning dépend **principalement de la qualité des données** :

* Nettoyer et organiser les données est souvent la tâche la plus longue et la plus critique.
* La mise en œuvre de l’algorithme est généralement simple grâce aux bibliothèques Python comme **Scikit-learn**.
* Développer de nouveaux algorithmes est rarement nécessaire : la plupart des solutions sont déjà disponibles et bien documentées.

> En résumé : la réussite d’un projet de machine learning repose sur **de bonnes données**, pas uniquement sur l’algorithme.
